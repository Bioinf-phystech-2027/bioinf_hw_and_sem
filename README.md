Конкретное значение E-value в контексте размера базы данных nr означает ожидаемое количество случайных совпадений с таким же или лучшим качеством выравнивания, которое может быть найдено при случайном переборе всей базы данных заданного размера . Raw Score нельзя использовать напрямую для оценки значимости, потому что Raw Score не учитывает длину последовательности и размер базы данных. Короткое, но очень точное совпадение может иметь низкий Raw Score, но при этом быть биологически значимым, тогда как длинное, но плохое совпадение наберёт больше баллов просто за счёт длины. Также Raw Score привязан к конкретной матрице замен.

### Задание 2. Поиск транслированных последовательностей.
Есть мРНК бета-глобина человека. Воспользуемся сервисом blastx (потому что он работает с translates nucleotides - proteins). blastx — переводит изучаемую нуклеотидную последовательность в кодируемые аминокислоты, а затем сравнивает её с имеющейся базой данных аминокислотных последовательностей белков, что идеально подходит для нашей задачи. Запустим по базе данных Swissprot.
![Результаты поиска blastx](blastx.png)
Как видно из скрина рамка считывания +3 оказалась верной для выравнивания с лучшим найденным белком.
![Лучший результат](best_score.png)
Подход использования сервиса blastx надежнее, чем через сторонние программы-трансляторы (хорошо хоть не на диваны-трансляторы), поскольку сервис blastx сможет учесть все рамки считывания в последовательности и выбрать ту, что будет иметь наилучший результат. При этом сторонние программы-трансляторы не будут устойчивы к ошибкам секвенирования и мутациям сдвига рамки считывания. Если мы переведем такую последовательность сами, то наш белок после мутации будет полностью неправильным и blastp ничего не найдет.
